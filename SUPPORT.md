# Support

Use this guide to choose the right support path for AiSH.

## Bugs

Open a bug report when AiSH crashes, fails to install, creates an incorrect shell profile, fails a release workflow, or behaves differently from the documented behavior.

Include:

- Operating system and version.
- Shell or terminal: PowerShell, Windows Terminal, VS Code terminal, bash, zsh, Terminal.app, etc.
- AiSH version or commit SHA.
- Install method.
- Exact command run.
- Expected behavior.
- Actual behavior.
- Logs or screenshots with secrets removed.

## Feature requests

Open a feature request for new shell behavior, model support, terminal integration, release packaging, website/download improvements, or developer experience improvements.

Describe the use case first, then the proposed implementation.

## Security issues

Do not use public issues for vulnerabilities. Follow `SECURITY.md`.

## Questions

Use GitHub issues only when the answer should become part of the project record. For transient questions, prefer direct maintainer discussion.

## Maintainer response expectations

AiSH is an early Dawnlight Labs pilot project. Response times may vary. High-impact installer, release, and shell safety issues should be prioritized over cosmetic issues.
