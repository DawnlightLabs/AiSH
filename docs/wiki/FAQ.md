# FAQ

## Is AiSH the old desktop app?

No. The active `main` branch is focused on the provider shell. The old Tauri desktop app is archived on the `app-provider-archive` branch.

## Does AiSH replace my terminal?

AiSH is designed to work with real terminal workflows. It should not hide generated commands or bypass shell behavior.

## Does AiSH require a cloud model?

AiSH is designed to support local-first behavior where possible, including local model path configuration. Exact model support depends on the current release.

## Can AiSH run destructive commands automatically?

Destructive or system-impacting commands should require explicit approval.

## Where do I report bugs?

Use the GitHub bug report template.

## Where do I report vulnerabilities?

Follow `SECURITY.md`. Do not post vulnerability details in public issues.

## Where are downloads?

Use the install commands in the README or download artifacts from GitHub Releases.
