# AiSH v0.4.6

This release makes local command planning substantially more reliable for the bundled Qwen2.5 Coder 1.5B model.

## Fixed

- Simplified constrained model output from five required fields to three: `action_type`, `command`, and `fallback_message`.
- Kept compatibility with the older five-field command-card format.
- Moved risk classification and approval decisions fully to AiSH's host-side safety layer.
- Resolved common navigation requests such as `go to downloads`, Desktop, Documents, home, and parent-directory navigation without invoking the model.
- Routed `aish --version` and `aish -v` as literal AiSH commands.
- Added lowercase `-v` version handling.

## Upgrade

Inside AiSH, run:

```text
/update
```

Or from PowerShell, Command Prompt, bash, or zsh:

```text
aish --update
```

Restart AiSH after the updater finishes.

## Validation

The release passed formatting, provider and provider-shell tests, workspace checks on Ubuntu and Windows, PowerShell syntax validation, website build, and Windows installer/updater lifecycle tests.
