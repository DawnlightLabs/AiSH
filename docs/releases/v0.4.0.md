# AiSH 0.4.0 — Branded Windows release

AiSH 0.4.0 completes the Windows application branding and cleans up terminal startup.

## Windows branding

- Embeds the approved AiSH app logo directly in `aish.exe`.
- Uses the branded executable icon in Windows Search, the Start menu, File Explorer, and Installed apps.
- Installs a versioned AiSH PNG for the Windows Terminal profile.
- Updates existing Windows Terminal profiles during installation and after an in-place update.
- Refreshes the Start menu shortcut and Windows icon cache during registration repair.

## Startup cleanup

- Stops the Windows registry registration probe from printing the entire `HKEY_CURRENT_USER` uninstall entry into every AiSH terminal session.
- Adds a Windows startup smoke test that fails if registry output leaks into the shell banner.

## Release automation

- Rebuilt the release workflow so all platform binaries are built and verified before a GitHub Release is published.
- Verifies that Windows release binaries contain the AiSH icon resource.
- Verifies the complete set of archives, checksums, and installer scripts after publishing.
- The Release Version action now bumps, tags, builds, and publishes in one flow.
- A `release/vX.Y.Z` branch can trigger the same release flow without temporary workflow files.

## Updating

Users on AiSH 0.3.1 can update normally:

```powershell
aish --update
```

After the update completes, reopen Windows Terminal. The AiSH profile and Start menu entry will use the approved AiSH logo.
