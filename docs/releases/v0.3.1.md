# AiSH 0.3.1 — Windows updater repair

AiSH 0.3.1 repairs the Windows update and application-registration paths introduced in the first stable release.

- Fixed Windows ZIP extraction failing with `Expand-Archive` because the archive and destination arguments were not reaching PowerShell.
- Fixed Windows Start menu and Installed apps registration using the same unreliable PowerShell argument pattern.
- Registration now runs through a parameterized PowerShell script with explicit named arguments.
- Added Windows CI smoke tests for archive extraction with spaces in paths, Installed apps registration, and uninstall cleanup.
- Future updates from AiSH 0.3.1 and newer can be installed through the normal automatic prompt, `/update`, or `aish --update`.

## Upgrade from 0.2.x or 0.3.0 on Windows

The faulty updater code is inside the already-installed binary, so affected installations must use the direct installer once:

```powershell
irm https://github.com/DawnlightLabs/AiSH/releases/latest/download/install.ps1 | iex
```

After that one repair installation, confirm the version:

```powershell
aish --version
```

Future releases will be detected and applied through AiSH itself.

## Uninstall

```powershell
aish --uninstall
```
