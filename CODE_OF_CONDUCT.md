# Code of Conduct

## Our standard

AiSH contributors are expected to keep project discussions practical, respectful, and focused on the work.

Examples of acceptable behavior:

- Using direct but professional language.
- Giving actionable feedback.
- Respecting different skill levels and operating systems.
- Keeping issue threads focused on reproduction, diagnosis, and resolution.
- Assuming good intent unless there is clear evidence otherwise.

Examples of unacceptable behavior:

- Harassment, insults, threats, or personal attacks.
- Discriminatory language or behavior.
- Publishing private information without permission.
- Repeatedly derailing technical discussions.
- Pressuring maintainers to bypass review, security, or release checks.

## Scope

This code applies in GitHub issues, pull requests, discussions, releases, project documentation, and other official AiSH project spaces.

## Enforcement

Maintainers may remove comments, close issues, block users, or restrict participation when behavior harms the project or its contributors.

Reports should be sent privately to a maintainer when possible. Include links, screenshots, and context. Do not publish private personal information in public issues.

## Attribution

This policy is adapted for the needs of the AiSH project and Dawnlight Labs.
